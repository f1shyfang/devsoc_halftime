## Affected Pages/Routes

- `/` (lobby) — fake credit balance display (localStorage), "Find Opponent" button, queue/cancel state, sessionId initialization
- `/debate/[id]` — Daily.co video iframe, turn timer (45s), audio recording controls per turn, transcript-after-Whisper display, turn-state UI
- `/result/[id]` — winner reveal, credit delta animation, shareable URL
- `/admin/seed` — auth-gated debug endpoint (header `x-admin-secret`), seeds room with pre-transcribed turns
- API: `/api/join-queue`, `/api/submit-turn`, `/api/transcribe`, `/api/judge`, `/api/prompt` (internal), `/api/pusher/auth` is REMOVED (presence cut)

## Key Interactions to Verify

- **Lobby → Find Opponent**: sessionId stable across page reloads (localStorage UUID generated once)
- **Matchmaking**: two browsers click "Find Opponent" within ~500ms — exactly one room is created, both clients receive Pusher `matched` event with the same `roomId`
- **Daily video joins**: both players see each other's video tiles in the iframe within ~3s of room load; mic permissions prompt fires once
- **Turn recording**: clicking "Start Turn" begins MediaRecorder on the mic stream; 45s timer counts down; clicking "Submit" stops recording, POSTs audio blob to `/api/transcribe`; transcript appears in transcript list
- **Auto-stop on timer expiry**: if user doesn't click submit, recording auto-stops at 0s, audio still posts, transcript appears
- **Judgment streaming**: clicking "Request Judgment" returns immediately (waitUntil pattern); reasoning prose streams in via Pusher buffered chunks (~80ms cadence); structured verdict revealed when `<VERDICT>...</VERDICT>` block lands
- **Credits transfer**: localStorage credit balance on each device updates correctly based on winner/loser role (winner +25, loser -25)
- **Share result**: copying URL from /result/[id] and opening in third browser shows the same verdict (server-stored)

## Edge Cases

- **Mic permission denied** on /debate/[id] → clear error UI, fallback "use text input" or explicit retry button
- **Two simultaneous joins** to matchmaking — atomic LPOP must produce one room, not two; second-place joiner waits or is matched with next entrant
- **Submit turn while not in `in_progress` state** (e.g., after timer expiry already auto-submitted) → 409 idempotent, no duplicate row
- **Malformed Claude JSON in verdict** → Zod parse fails → server regenerates ONCE with strict prompt → if still malformed → return draw verdict, refund stakes
- **Claude API timeout / down** for /api/prompt → fallback to one of 3 hardcoded prompts (no error to user)
- **Whisper API timeout** for /api/transcribe → clear error UI, allow retry, do not advance turn
- **Daily.co video fails to load** → /debate/[id] still functional for typed/recorded turns (audio is the source of truth for transcription; video is presence-only)
- **Browser refresh mid-debate** → on /debate/[id] reload, sessionId in localStorage matches a player in the room → resume current turn state from server
- **Duplicate /api/judge call** (two players click "Request Judgment" near-simultaneously) → SETNX lock, second caller gets 409, both clients still receive Pusher events from the first
- **Pusher streaming chunk overflow** → buffered flush every ~80ms prevents trigger storm; verify total triggers per judgment stays under 25
- **localStorage credits go negative** → UI floors at 0, "out of credits" message, prevents joining queue (post-hackathon: server-side credit truth)
- **/admin/seed without correct header** → 401 immediately, no Redis writes
- **/admin/seed with valid header** → seeds room with `state=complete` and pre-transcribed turns, ready for direct /api/judge invocation (debug path)

## Critical Paths (must work end-to-end)

1. **Live debate happy path**: Open lobby on two devices → both click "Find Opponent" → matched → see each other's video → both speak 3 turns each (transcribed via Whisper) → click "Request Judgment" → verdict streams in → winner gets credits, loser loses credits → result card loads → share URL works
2. **Demo fallback path**: /admin/seed creates a room with pre-transcribed turns → second device opens /debate/[seeded-id] → clicks "Request Judgment" → verdict streams in → result card loads. This is the bypass-the-audio-stack demo backup when stage mic conditions are uncertain.
3. **Concurrent matchmaking under load**: 4+ devices all clicking "Find Opponent" within 1 second → exactly 2 rooms created, all devices end up in a room (no orphans, no double-matched)
4. **Whisper failure recovery**: if /api/transcribe fails for a turn, the player can retry that turn without losing the room or skipping ahead

## Unit Tests (the 4-test smoke kit from D12)

Located at `app/__tests__/` or similar, runnable via `vitest`:

1. **`verdict-parser.test.ts`** — Zod schema validation: valid `<VERDICT>{...}</VERDICT>` parses; trailing prose ignored; malformed JSON triggers retry path; double-fail returns draw verdict; field types validated (winner enum, scores 1-10).
2. **`matchmaking.test.ts`** — Mock Upstash client: empty queue → join returns "waiting"; 1 in queue + 1 joining → atomic LPOP returns both, room created; 3 in queue → exactly 2 popped, third remains.
3. **`turn-dedup.test.ts`** — Mock Redis: same `turn_index` POSTed twice → second is idempotent 200, no double row in `room:{id}:turns`.
4. **`state-guard.test.ts`** — submit-turn in `complete` state → 409; submit-turn in `in_progress` → 200; judge in `in_progress` → 409 (not yet complete); judge in `complete` → 200.

Skipped (manual coverage via /admin/seed + night-before two-device run):
- E2E browser tests
- Daily.co integration tests (managed service, trust it)
- Whisper integration tests (managed service, manual smoke test sufficient)
- Eval suite for judge prompt quality (defer to post-hackathon)
